# 真机部署 + 父母走查清单

目标：把当前服务暴露到父母的手机上，本人先当父母走一遍，验证"父母到底用不用"（R-04）。
MVP 阶段**不上云**，用「家里 PC 跑服务 + 内网穿透」最省事省钱。

---

## 一、为什么必须用 HTTPS 内网穿透（关键）

手机浏览器对"安全环境（secure context）"有硬性要求：

| 功能 | HTTP（局域网 IP） | HTTPS（穿透域名） |
|---|---|---|
| 看首页 / 农事 / 预警 | ✅ | ✅ |
| 📷 拍照诊断 | ✅（文件选择器，HTTP 也行） | ✅ |
| 🎤 语音输入（ASR） | ❌ 不可用 | ✅ |
| ➕ 添加到主屏幕（PWA） | ❌ 受限 | ✅ |
| 离线壳（Service Worker） | ❌ | ✅ |

> 结论：**直接用 `http://192.168.x.x:8770` 会让语音和"装到桌面"失效。** 用带 https 的内网穿透一次解决可达性 + 安全环境。

## 二、推荐方案：cpolar（国内友好，免费档够用）

### 1. 本机启动服务
```powershell
cd D:\claude_code\20260606_ai赋农\try_ai_by_apple
python server.py
# 启动后控制台会打印可访问地址
```

### 2. 安装并启动 cpolar 隧道
- 到 https://www.cpolar.com 注册、下载、装好并 authtoken 登录。
- 把本机 8770 端口暴露为 https：
```powershell
cpolar http 8770
```
- 终端会显示一个形如 `https://xxxx.r6.cpolar.top` 的公网地址（**https** 的那条）。

### 3. 手机访问
- 把这个 https 地址用微信/浏览器打开（可生成二维码让父母扫）。
- 在浏览器菜单选「添加到主屏幕」，桌面会出现绿色「桃」图标，以后点图标像 App 一样打开。

> 备选穿透工具：ngrok（海外）、frp（自建服务器，需自己配 TLS）、ZeroTier（组虚拟局域网，但仍是 http，不解决 HTTPS）。**优先选能直接给 https 域名的。**

## 三、局域网直连（仅快速自测，不推荐给父母）

只想在同一 WiFi 的手机上快速看页面（不测语音）时：
```powershell
python server.py --host 0.0.0.0
# 控制台会打印 http://<本机IP>:8770/
```
手机连同一 WiFi 打开该地址即可。**注意：此方式语音输入和"添加到主屏幕"不可用。**

## 四、父母走查清单（R-04 验证）

部署好后，**本人先扮父母走一遍**，逐项记录卡点。重点不是功能对不对，而是**老人能不能独立完成、看不看得懂**。

- [ ] 打开 https 地址 → 能否顺利「添加到主屏幕」？图标/名字对不对？
- [ ] 点桌面图标进入 → 首页一屏内能否看清：今天日期、当前是什么管理期、本周该做什么？
- [ ] 字够不够大？按钮够不够好点？太阳下/老花镜下看得清吗？
- [ ] 📷 拍照诊断：点一下 → 能否唤起相机/相册 → 拍一张叶片 → 看懂结果吗？用药那块"导向农技员"会不会让父母困惑？
- [ ] 🔊 听：点"听"按钮 → 声音大小、语速、口音能听懂吗？
- [ ] 🎤 问问 AI：按住说话（陕西话）→ 识别准不准？识别错时父母会不会用打字兜底？
- [ ] 🌨️ 预警（用 `?scenario=hail` 演示）：横幅够不够醒目？点开能看懂"来之前/发生时/过后"吗？
- [ ] 整体：父母**不需要你在旁边**能独立用吗？哪一步卡住了？
