# 风险登记 — SSOT

> 定期回看。每条含：风险 / 影响 / 概率 / 对策 / 状态。
> 等级：🔴高 🟡中 🟢低。
> 版本：0.1.0

---

## R-01 🔴 用药建议错误致损
- **影响**：错误用药 → 药害/减产/农残超标/食品安全，最严重的风险。
- **对策**：用药只走人工查录的「已登记农药表」，AI 不生成（ADR-010 / 知识治理§4）；常驻免责声明；拿不准导向农技员。
- **状态**：对策已定，待 `pesticide_kb.jsonl` 落地。

## R-02 🔴 极端天气漏报（霜冻/冰雹）
- **影响**：花期霜冻漏报 = 整季绝收。
- **对策**：硬规则阈值兜底，不依赖 LLM（ADR-009）；多次推送 + 语音；预警提前量足够。
- **状态**：对策已定，待 `alert_scheduler` 实现。

## R-03 🔴 单点维护风险（你一个人）
- **影响**：维护者没空 → 项目停摆 → 父母用着用着就没了。
- **对策**：P0 砍到极致先上线；知识/配置数据化（改内容不改代码）；SSOT 文档化，便于将来捡起。
- **状态**：持续——每次加功能都要问"这会不会让我更难维护"。

## R-04 🟡 父母不用 / 不信任（产品定位风险）
- **影响**：做得再准，父母不主动用 = 失败。
- **对策**：定位为"子女工具+父母被动接收"（IA§0）；推送驱动；首次当面演示建立信任；适老化硬约束。
- **度量**：北极星=父母**主动**使用。3 个月只有你在用 → 定位需回调。
- **状态**：靠 P0 上线后观察验证。

## R-05 🟡 AI 幻觉/误诊
- **影响**：诊断错 → 误导处置。
- **对策**：诊断分"确定/可能/拿不准"三档；拿不准导向补拍或农技员；RAG 带出处；反馈闭环修正。
- **状态**：prompt 层已写（tasks.py），待 RAG + 置信度表达落地。

## R-06 🟡 成本失控
- **影响**：vision/TTS/天气 API 调用费 + 服务器，长期自费难持续。
- **对策**：天气缓存、TTS 缓存、vision 按需调用；MVP 内网穿透省服务器；监控调用量。
- **状态**：缓存策略待实现。

## R-07 🟡 农村网络差 / 服务中断
- **影响**：信号差或 AI 服务挂 → 老人面对报错会慌、会弃用。
- **对策**：PWA 离线壳缓存今日农事/本周建议/最近诊断；每个 AI 功能有"不依赖 AI"的保底版；失败话术友好（"网不好待会儿再试"）。
- **状态**：降级链待实现。

## R-08 🟡 知识无人审核（当前阶段）
- **影响**：AI 草拟知识可能有误，暂无农技员把关。
- **对策**：无审核员期保守模式（知识治理§6）；用药表人工查录；反馈闭环+子女判断临时替代；待审队列等将来批量校验。
- **状态**：临时治理已定，靠时间积累审核员。

## R-09 🟢 隐私/数据合规
- **影响**：拍照、地块经纬度等敏感数据处理不当。
- **对策**：照片默认不留存（config `keep_uploaded_images_days:0`）；位置本地存不上传第三方；账户用微信 openid 最轻量。
- **状态**：config 已留默认，待实现时遵守。

## R-10 🟢 方言识别率低
- **影响**：关中话语音转文字不准，影响问答体验。
- **对策**：通用 ASR + 按钮兜底（ADR-007），语音是加分项非唯一项。
- **状态**：已接受，靠交互设计兜底。

## R-11 🟢 多作物扩展时重构
- **影响**：硬编码 peach → 扩苹果/梨/樱桃要大改。
- **对策**：day-1 带 `crop` 维度（ADR-004）。
- **状态**：已规避。
