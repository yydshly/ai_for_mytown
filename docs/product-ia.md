> 版本：0.1.0 ｜ 适用作物：桃树（MVP），数据模型预留多作物 `crop` 维度。

---

## 0. 产品定位（最重要的一句）

**这不是农业 SaaS，是"子女远程协助父母种地"的家庭产品。**

- 每天主动打开 App、配置、兜底的是 **子女（caretaker）**。
- 父母（farmer）是被**推送 + 语音**找上门的一方，只需要看大字、点大按钮、听朗读。
- 设计任何功能时自问：父母这一端能不能"零学习、被动接收"？子女这一端能不能"远程代办"？

## 2. 移动端 H5 信息架构（IA）

老人产品金律：**底栏 ≤4 tab，首页大卡片，关键操作 ≤3 步。**

```
顶部：今日 · 关中 · 果实膨大期
┌────────────────────────────────────┐
│ 🌦 [预警横幅，有则显示]            │
│   "明早有霜冻，点这里看怎么办"      │
├────────────────────────────────────┤
│ 📸 拍照诊断 │ 🎤 问问AI │ 📋 今日农事 │  ← 三大快捷入口
├────────────────────────────────────┤
│ 本周建议（AI 综合天气+物候 4-5 条）│
└────────────────────────────────────┘
底栏：[首页] [农事] [我的园子] [我]
```

| Tab | 包含模块 | 主要用户 |
|---|---|---|
| 🏠 首页 | F1/F2 推送、B1 天气、B2 预警、本周建议、三大快捷入口 | **父母**（只看这一屏就够） |
| 📅 农事 | A3-A7 种植指导（按物候期组织）、B5 用药记录入口 | 双方 |
| 🌳 我的园子 | D1 地块、D2 日志、C 采后销售、D3 账本 | **子女**（配置、翻看） |
| 👤 我 | F3 家人绑定、F4 通讯录、E3 知识库、E4 专家、设置 | **子女**（配置） |

> 父母端默认只呈现"首页"；其余 tab 主要供子女配置和查看。

## 3. 双角色场景流

```
父母（陕西，地里）        微信推送+语音           子女（外地）
   │  ◄──────────"明早霜冻，地块A花期，需熏烟"──────────►  │
   │  点开→大字方案→听语音                              │ 同时收到推送
   │  拍照诊断→看结果                                   │ 后台帮父母配置/确认
   │  不放心→"找专家"一键拨号                           │ 群里看到诊断，必要时联系农技员
```

**核心结论：一个产品，两类用户，两套界面密度。** 数据模型 day-1 即带 `farmer_id` + `caretaker_id` 绑定关系。

## 4. MVP（P0）范围 — 已确认

第一版只做以下闭环，其余进 P1/P2：

1. **首页框架**（大卡片 + 三快捷入口 + 预警横幅）
2. **农事日历**（A3/A4/A6，按物候期，种子数据已建，待父母校准）
3. **天气 B1 + 灾害预警 B2**（硬规则兜底）
4. **拍照诊断 B3+B4**（vision + RAG）
5. **语音/文字问答 E2**
6. **推送 F1 + 语音播报 F2**
7. **双账户最小版 F3**（父母看推送、子女配地块）
8. **地块档案 D1 最小版**（位置/品种，给天气和物候定位用）

P0 的北极星：**父母是否"主动"使用**（详见 risks.md / 成功度量）。

## 5. 后端领域划分（按业务组织，非按文件类型）

```
src/backend/domain/        领域模型与业务规则
  growing/      A1-A7 种植指导（含物候期 phenology）
  protection/   B1-B5 防护（天气、灾害、病虫、用药）
  postharvest/  C1-C5 采后与销售
  records/      D1-D4 档案（地块、日志、账本、库存）
  knowledge/    E1-E3 AI 任务 + 知识库 + RAG（见 knowledge-governance.md）
  collab/       F1-F4 通知、协同、通讯录
  identity/     用户 + 角色（farmer/caretaker/农技员）+ 双账户绑定
```

> 注：当前 Phase 1 代码仍是扁平 `routes/services`，随 P0 推进逐步收敛到上述领域边界。
> 所有领域数据**必带 `crop` 字段**，桃树只是第一个值。
